# RJChicago, LLC - Consulting Website

A modern, responsive single-page application built with Next.js and Tailwind CSS for RJChicago, LLC consulting business.

## Features

- **Single-page layout** with smooth scrolling navigation
- **Fully responsive** design optimized for all devices
- **Accessible** with proper ARIA labels, keyboard navigation, and semantic HTML
- **SEO optimized** with meta tags and Open Graph data
- **Configurable content** - all text, pricing, and contact info in one file
- **Working contact form** with validation and configurable submission method
- **Modern design** with clean typography using Inter font
- **Performance optimized** with minimal dependencies

## Sections

- **Navigation**: Sticky header with smooth scroll links
- **Hero**: Compelling headline with call-to-action buttons
- **Services**: Cards showcasing SRE, Kubernetes, DevOps, Cloud, and AI services
- **Pricing**: Hourly rates and fixed-fee packages
- **About**: Company credibility and background
- **Contact**: Working form with validation
- **Footer**: Company info and links

## Quick Start

### Frontend (`web/`)
1. ```bash
   cd web
   npm install
   npm run dev
   ```
2. Visit `http://localhost:3000`.

### Backend (`api/`)
1. ```bash
   cd api
   npm install
   cp .env.example .env # update values
   npm run dev
   ```
2. The API listens on `http://localhost:4000` by default.

## Configuration

All marketing copy lives in `web/src/config/siteConfig.js`. Update this file to customize:

- Company information and branding
- Contact details and form settings
- Service descriptions and pricing
- SEO meta tags and social media data
- Navigation links and footer content

### Key Configuration Areas

**Contact Form Setup**:
```javascript
contact: {
  email: "your@email.com",           // Update with your email
  formEndpoint: process.env.NEXT_PUBLIC_CONTACT_ENDPOINT || "http://localhost:4000/contact",
  useMailto: false,                  // Switch to true for mailto fallback
}
```
Set `NEXT_PUBLIC_CONTACT_ENDPOINT` in `web/.env.local` when targeting a non-local backend.

**Pricing Updates**:
```javascript
pricing: {
  hourlyRates: [
    { rate: "$150", ... },           // Update rates
    { rate: "$175", ... }
  ],
  packages: [
    { price: "$3,500", ... },        // Update package prices
  ]
}
```

## Customization

### Styling
- Built with Tailwind CSS for easy customization
- Custom color scheme defined in `tailwind.config.js`
- Global styles and utility classes in `web/src/styles/globals.css`

### Contact Form
- Default setup posts to the local API service at `http://localhost:4000/contact`.
- Switch back to the mail client flow by toggling `useMailto` and `formEndpoint` in `siteConfig`.
- When posting to the API, include a `captchaToken` field generated by your CAPTCHA provider (e.g., Cloudflare Turnstile). Configure the backend with `CAPTCHA_SECRET` (and optionally `CAPTCHA_VERIFY_ENDPOINT`) so the API can verify the signed token before sending email. If `CAPTCHA_SECRET` is omitted (e.g., for local development), the API skips verification and the token becomes optional.

### Adding Content
To add new sections or modify existing ones:

1. Update `web/src/config/siteConfig.js` with new content
2. Create new components in `web/src/components/` directory
3. Import and add to `web/src/pages/index.js`

## Build and Deploy

### Local Development
1. **Build for production**:
   ```bash
   npm run build
   ```

2. **Start production server**:
   ```bash
   npm start
   ```

### Docker Deployment
- `docker-compose up rjchicago-dev --build` starts the frontend dev server.
- `docker-compose up rjchicago-api --build` starts the backend.
- `docker-compose up` (no service name) brings both up together; add `-d` for detached mode.
- `docker-compose down` stops everything.

### Platform Deploy
The built application can be deployed to any hosting platform that supports Node.js (Vercel, Netlify, AWS, etc.)
